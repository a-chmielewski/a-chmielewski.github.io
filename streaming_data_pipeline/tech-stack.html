Technology Overview

Technology Stack

A comprehensive look at the technologies, libraries, and tools that power this production-ready streaming data pipeline.

Core Technologies

The foundational tools that make real-time streaming possible

üêç Python 3.9+

Role: Primary programming language for WebSocket producer

Why chosen:

  • Rich ecosystem for data engineering (pandas, kafka-python)
  • Excellent WebSocket library support
  • Easy JSON handling and data manipulation
  • Quick prototyping and development
  • Wide adoption in data engineering community

Key libraries:

  • websocket-client - WebSocket connection handling
  • kafka-python - Kafka producer client
  • json - Message parsing and serialization

üì® Apache Kafka

Role: Distributed message queue and streaming platform

Why chosen:

  • Industry standard for real-time data pipelines
  • Provides durability with disk-based persistence
  • Enables decoupling between producers and consumers
  • Handles backpressure automatically
  • Supports horizontal scaling via partitions

Configuration:

  • Topic: crypto-trades
  • Partitions: 1 (scalable to multiple)
  • Retention: 7 days
  • Replication factor: 1 (single-node deployment)

‚ö° Apache Spark 3.5+

Role: Stream processing engine

Why chosen:

  • Structured Streaming API for SQL-like operations
  • Exactly-once processing semantics
  • Native Kafka integration
  • Micro-batch processing for efficiency
  • Easy integration with Elasticsearch

Components used:

  • spark-sql-kafka-0-10 - Kafka source connector
  • elasticsearch-spark - Elasticsearch sink connector
  • DataFrame API for transformations
  • Structured Streaming for continuous processing

üîç Elasticsearch 8.x

Role: Search and analytics engine for data storage

Why chosen:

  • Optimized for time-series data
  • Fast full-text search and aggregations
  • Horizontal scalability with sharding
  • RESTful API for easy integration
  • Native Kibana integration

Index configuration:

  • Index name: crypto-trades-v2
  • Shards: 1, Replicas: 0 (development setup)
  • Field mappings: date, keyword, double, boolean
  • Optimized for timestamp-based queries

üìä Kibana 8.x

Role: Data visualization and exploration

Why chosen:

  • Purpose-built for Elasticsearch
  • Powerful dashboard creation tools
  • Real-time data refresh capabilities
  • KQL (Kibana Query Language) for filtering
  • Multiple visualization types out-of-the-box

Features utilized:

  • Time-series line charts
  • Aggregation-based metrics
  • Pie charts and histograms
  • Auto-refresh dashboards
  • Interactive filtering

Data Source

Where the data comes from

üåê Binance WebSocket API

Endpoint: wss://fstream.binance.com/ws/[symbol]@aggTrade

What it provides:

  • Real-time aggregate trade events from Binance futures
  • Sub-second latency for market data
  • JSON-formatted messages
  • Free tier available (no API key required for public streams)
  • Reliable WebSocket connection with heartbeat

Message format:

{
  "e": "aggTrade",
  "E": 1730638245123,
  "s": "BTCUSDT",
  "a": 1234567890,
  "p": "107572.10",
  "q": "0.045",
  "f": 100,
  "l": 105,
  "T": 1730638245100,
  "m": false
}

Supported symbols: Any Binance futures trading pair (BTCUSDT, ETHUSDT, SOLUSDT, etc.)

Infrastructure & DevOps

Tools for deployment and orchestration

üê≥ Docker

Role: Containerization platform

Usage:

  • Producer application containerized with Python base image
  • All services run in isolated containers
  • Consistent environment across development and production
  • Easy dependency management

Dockerfile highlights:

FROM python:3.9-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY producer/ .
CMD ["python", "websocket_producer.py"]

üêô Docker Compose

Role: Multi-container orchestration

Services managed:

  • kafka - Message broker (port 9092)
  • elasticsearch - Data storage (port 9200)
  • kibana - Visualization (port 5601)
  • producer - WebSocket client
  • spark-streaming - Stream processor

Features:

  • Dependency management (depends_on)
  • Environment variable injection
  • Network isolation and service discovery
  • Volume persistence for data
  • Health checks and restart policies

Performance Characteristics

Expected performance and resource usage

Latency

  • WebSocket to Kafka: < 100ms
  • Kafka to Spark: 10s (micro-batch interval)
  • Spark to Elasticsearch: < 1s
  • End-to-end: ~11-12 seconds

Throughput

  • WebSocket: 1000+ messages/sec
  • Kafka: 10,000+ messages/sec
  • Spark: 5,000+ records/sec
  • Elasticsearch: 3,000+ docs/sec

Resource Usage

  • Producer: ~100MB RAM, <1% CPU
  • Kafka: ~512MB RAM, 5-10% CPU
  • Spark: ~2GB RAM, 10-20% CPU
  • Elasticsearch: ~2GB RAM, 10-15% CPU
  • Kibana: ~1GB RAM, 5% CPU

Scaling Limits

  • Single-node: 10K events/sec
  • With tuning: 50K events/sec
  • Multi-node cluster: 100K+ events/sec
  • Storage: Hundreds of GB per day
