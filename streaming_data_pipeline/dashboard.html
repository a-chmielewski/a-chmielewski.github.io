Metrics Explained

Understanding the dashboard data points

Price Metrics

What it shows: Current, minimum, maximum, and average prices over the selected time range.

Why it matters: Helps identify price trends, volatility, and trading ranges. Large price swings indicate high volatility.

Calculation: Aggregated from individual trade prices using Elasticsearch aggregations (avg, min, max).

Trade Volume

What it shows: Total quantity of cryptocurrency traded in each time bucket.

Why it matters: Volume indicates market activity and liquidity. High volume with price movement confirms trend strength.

Calculation: Sum of the "quantity" field across all trades in the time window.

Buy vs Sell Ratio

What it shows: Percentage split between buyer-initiated (market buy) and seller-initiated (market sell) trades.

Why it matters: When buyers > 50%, indicates bullish sentiment. When sellers > 50%, indicates bearish pressure.

Calculation: Count trades where "is_buyer_maker" is false (buy) vs true (sell), then compute percentage.

Trade Count

What it shows: Number of individual trades executed per time interval.

Why it matters: High trade count indicates active market participation. Spikes often correlate with news events.

Calculation: Simple count of documents in Elasticsearch grouped by time buckets.

Price Distribution

What it shows: Histogram of trade frequency at different price levels.

Why it matters: Peaks in the histogram represent price levels where most trading occurred (support/resistance).

Calculation: Histogram aggregation with price buckets (e.g., $100 intervals).

Sample Queries

Useful KQL queries for data exploration

// Find all trades above $108,000
price > 108000

// Show only buyer-initiated trades
is_buyer_maker: false

// Large trades (> 0.5 BTC)
quantity > 0.5

// Trades in specific price range
price >= 107000 AND price <= 108000

// Recent trades (last 5 minutes)
timestamp >= "now-5m"

// High-volume trades
quantity > 0.3 AND price > 100000

Pro Tip: Combine multiple filters using AND/OR operators. Save frequently used queries as filters for quick access. Use the time picker in combination with KQL for powerful analysis.

Creating Custom Visualizations

Build your own charts and metrics

1. Choose Visualization Type

Select from line charts, bar charts, pie charts, data tables, metric displays, heatmaps, and more.

2. Select Index Pattern

Use the "crypto-trades-v2" index pattern to access all trade data fields.

3. Configure Aggregations

Set up metrics (avg, sum, count, min, max) and buckets (date histogram, terms, ranges).

4. Apply Filters

Add KQL filters to focus on specific symbols, price ranges, or time periods.

5. Customize Appearance

Adjust colors, labels, legends, and axis settings to match your preferences.

6. Add to Dashboard

Save the visualization and add it to an existing dashboard or create a new one.

Accessing the Dashboard

How to view the live dashboard in your deployment

Local Deployment Access

  1. Start the pipeline using docker compose up -d
  2. Wait for Elasticsearch and Kibana to be ready (~2 minutes)
  3. Navigate to http://localhost:5601 in your browser
  4. Login with username: elastic and password from your .env file
  5. Go to Analytics ‚Üí Dashboard in the navigation menu
  6. Create a new dashboard or import the pre-configured one

Note: The dashboard requires data to be flowing through the pipeline. Make sure the WebSocket producer and Spark streaming jobs are running. It may take 10-20 seconds for data to appear after starting the services.
