Design Philosophy: This pipeline implements a complete data quality testing framework from API ingestion to visualization. Raw data is persisted for reprocessability, quality checks run automatically, and all validation results are documented for audit trails. The Makefile-based orchestration ensures transparent, reproducible execution suitable for CI/CD integration.

Data Quality Checks

Comprehensive validation with Great Expectations

Schema Validation

expect_table_columns_to_match_set

Ensures all required columns exist in the dataset. Catches schema evolution and missing fields that could break downstream transformations.

Primary Key Constraint

expect_column_values_to_be_unique

Validates that unique_key column contains no duplicates. Essential for maintaining data integrity and preventing double-counting in analytics.

NOT NULL Checks

expect_column_values_to_not_be_null

Enforces mandatory fields including created_date and complaint_type. Ensures critical business dimensions are always populated for reporting.

Volume Validation

expect_table_row_count_to_be_between

Validates row counts fall within expected range (50-5000). Catches API failures, network issues, or unexpected data volume changes.

Enum Validation

expect_column_values_to_be_in_set

Ensures borough values match valid NYC boroughs (Manhattan, Brooklyn, Queens, Bronx, Staten Island) with 90% threshold for data quality tolerance.

Validation Results

Automated Reporting

Results stored in gx/uncommitted/validations/ with pass/fail status, statistics on unexpected values, and metadata for tracking validation history. Failed validations trigger pipeline exit codes for CI/CD integration.

Production-Ready Quality Gates: All expectations run automatically through checkpoints, generating detailed HTML reports (Data Docs) for stakeholder review. Failed validations exit with code 1, enabling CI/CD pipeline integration and automated alerting.

Pipeline Workflow

Five-stage execution orchestrated with Makefile

# Run complete pipeline
make all

# Individual stages
make fetch       # Data ingestion from API
make dbt         # dbt transformations
make gx          # Initialize Great Expectations (first run)
make checkpoint  # Run validation checks
make metrics     # Generate dashboard metrics

1. Data Ingestion (make fetch)

  • Query NYC Open Data API with date filters
  • Fetch last 7 days + 2019 baseline
  • Validate schemas with Pandera
  • Export to CSV in data/ directory
  • Load into DuckDB raw schema

2. dbt Transformations (make dbt)

  • Staging: Type casting & deduplication
  • Mart: Business logic & aggregations
  • Built-in data quality tests
  • Generate lineage documentation

3. GX Setup (make gx)

  • Initialize Great Expectations context
  • Configure Pandas datasource
  • Create expectation suites
  • Define validation checkpoint

4. Quality Validation (make checkpoint)

  • Load datasets into GX validators
  • Run all expectation suites
  • Generate HTML Data Docs reports
  • Exit with failure if checks fail

5. Metrics Generation (make metrics)

  • Compute row counts per dataset
  • Calculate null ratios for key columns
  • Identify top complaint types
  • Export to artifacts/metrics.json
