Design Philosophy: Raw data is persisted in MinIO for reprocessability and audit trails. Quality validation happens before database insertion, maintaining data integrity. Idempotent operations allow safe pipeline reruns without data duplication. Incremental processing fetches only data after the last successful load timestamp.

Data Model

Medallion architecture with staging and marts layers

Staging Layer

Table: staging.stg_weather_hourly

Grain: Hourly weather measurements per city

Keys: (city, timestamp)

  • Raw measurements from API
  • No transformations applied
  • Direct load from MinIO
  • Validated by Great Expectations

Mart Layer

Table: mart.fct_city_day

Grain: Daily aggregates per city

Keys: (city, date)

  • Temperature min/max/avg
  • Precipitation sum
  • Wind speed max
  • Optimized for BI queries

Data Quality Rules

Great Expectations Validations

  • Not null: city, timestamp
  • Temperature range: -60¬∞C to 60¬∞C
  • Precipitation: >= 0
  • No duplicates on (city, timestamp)
  • Schema validation

dbt Lineage

Transformation Flow

  • Source: staging.stg_weather_hourly
  • Transform: Daily aggregations
  • Target: mart.fct_city_day
  • Tests: Data quality checks
  • Docs: Auto-generated lineage
